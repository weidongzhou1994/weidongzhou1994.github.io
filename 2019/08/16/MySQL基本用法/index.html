  MySQL 是关系型数据库，用法相对较为复杂，首要的是设计数据库，即通过 E-R 模型设计实体和关系，包括表表关系，如 1 对 1、1 对多和多对多。    除此之外，MySQL 数据库还遵循三范式：列不可拆分、标志唯一和引用主键。 # 1. 数据库的完整性   MySQL 支持多种类型，大致可以分为三类：数值、日期 / 时间和字符串 (字符) 类型。    数值类型主要包括 INT 和 DEC，前者即为整数型，后者则更为强调精度。1    字符串类型主要包括 char - 定长字符串、varchar - 变长字符串和 text - 长文本数据。    日期和时间类型主要包括 datetime - 日期，除此之外还有布尔类型的数据 - bit。    完整性约束包括主键、非空、唯一、默认和外键，设置主键的常用语法如下：

9. 事务

  MySQL 事务主要用于处理操作量大，复杂度高的数据。比如说，在人员管理系统中，你删除一个人员，你即需要删除人员的基本资料，也要删除和该人员相关的信息，如信箱，文章等等，这样，这些数据库操作语句就构成一个事务！    一般来说，事务是必须满足 4 个条件（ACID）：：原子性（Atomicity，或称不可分割性）、一致性（Consistency）、隔离性（Isolation，又称独立性）、持久性（Durability）。
